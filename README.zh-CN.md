# Bit Slice

> [English](README.md)

Bit Slice 是一个轻量的浏览器工具，用来在 bit 级查看十六进制数据：可视化 bit、移动对齐窗口、搜索匹配模式，并把切片结果重新组装成 hex。

**在线演示**：[sure-wang.github.io/bit-slice](https://sure-wang.github.io/bit-slice/)

## 为什么做

在调 FPGA、嵌入式协议、寄存器 payload、报文 dump 或二进制日志时，真正麻烦的通常不是进制转换，而是这些问题：

- 我现在对齐到了第几个 bit？
- 这个窗口切出来对应什么 hex？
- 某个 Verilog 风格的匹配模式出现在哪里？
- 我是不是偏移了 1 bit？

Bit Slice 的目标是把这些答案直接放在眼前，同时不要求安装依赖、不需要启动工程，也不用把数据丢进复杂工具里。

## 项目 Taste

- **默认轻量**：一个静态 HTML 文件，方便 GitHub Pages 托管，也方便本地直接打开。
- **精确优先**：界面重点放在 bit 位置、窗口范围、匹配计数和当前值，而不是装饰。
- **反馈即时**：粘贴 hex、拖动起始 bit、调整宽度，结果立即更新。
- **数据可读**：bit 轻分组显示，匹配结果用浅蓝色，高亮窗口用黄色边框。
- **对新人友好**：内置示例，不读 README 也能快速理解基本流程。

## 功能

- **Hex 输入**：自动校验、转大写，最多 512 个 hex 字符。
- **Bit 视图**：按 4 bit 轻分组，显示当前窗口和匹配高亮。
- **切片窗口**：展示 `start`、`end`、`width`、`match`、`value` 状态。
- **模式匹配**：支持 Verilog 风格 hex（`N'hXX`）、普通 hex 和二进制字符串。
- **切片结果**：按窗口宽度重新组装为 hex，可复制当前切片或全部结果。
- **输入历史**：通过 `localStorage` 保存在本地浏览器。

## 使用方法

打开[在线演示](https://sure-wang.github.io/bit-slice/)或在浏览器中打开本地 `index.html`。

1. 粘贴或输入 hex 数据，例如 `5E4D6EADA`。
2. 拖动起始 bit 滑条，设置窗口宽度。
3. 可选：搜索 `3'h5`、`8'hB7`、`B7` 或 `10110111` 这类匹配模式。
4. 通过状态栏确认 `start`、`end`、`width`、`match` 和 `value`。
5. 需要时复制当前切片或完整切片结果。

## 适合场景

- FPGA 信号对齐和 `rxdata` 检查
- 嵌入式协议 payload 分析
- 寄存器字段和 packed-bit 数据检查
- 报文或二进制日志调试
- bit 顺序、切片和对齐概念教学

## 限制

- **输入上限**：最多 512 个 hex 字符，即 2048 bit。
- **窗口大小**：受输入长度限制；内部使用 `BigInt` 转换，在输入上限内不会丢精度。
- **数据保存**：历史记录只存在本地浏览器的 `localStorage`，不会上传数据。

## 开发

这个项目刻意保持简单：

- `index.html` 包含完整应用。
- `test.js` 包含独立逻辑测试。
- 不需要构建步骤。

运行测试：

```bash
node test.js
```

测试覆盖 hex/二进制转换、Verilog 风格解析、位序列匹配、切片重组和边界情况。
